SecureCorp Lab Guide

Work through each phase on your own. Click Hints only when you are stuck.

Setup

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
VULNLAB_RESET=1 python app.py

Open http://127.0.0.1:5000 and explore the portal.

Phase 1 — Reconnaissance

Goal: Map the application before attacking anything.

  1. Browse the site like a new employee. Note pages, forms, and links.
  2. Look beyond the UI — source code, machine-readable files, common paths.
  3. Find APIs, debug endpoints, or leaked configuration.
  4. Record usernames, fields, cookies, and error messages.

Done when: You have a list of interesting URLs and unexpected sensitive data.

Hints
Where to start looking

View source on the sign-in page. Fetch /robots.txt. Try /api/status for JSON.

Leaked configuration

robots.txt references a path under /backup/. Read the full file — it contains labeled values for later.

Phase 2 — Authentication & sessions

Goal: Get past login and reach administrator content.

  1. Try valid and invalid credentials. Compare error messages.
  2. Intercept the login request. Inspect every parameter sent.
  3. After login, examine cookies and session storage.
  4. Reach the administrator resources on the dashboard.

Done when: You can view the admin panel and recorded its labeled values.

Hints
Login interception

The sign-in POST includes hidden fields. Capture and edit the request in Burp, ZAP, or DevTools.

Bypassing authentication

Input is concatenated into a SQL query. Break out of the string in the username field. Combine with role tampering or cookie editing for admin access.

User enumeration

Failed logins return different errors for unknown vs. known usernames. A JSON endpoint accepts a username parameter.

Cookie tampering

The vulnlab_session cookie is URL-safe base64 JSON, not HttpOnly. Decode, set role to admin, re-encode, replace, reload.

Test credentials

Check the HTML comment on the sign-in page for a guest account.

Phase 3 — Cross-site scripting

Goal: Trigger reflected and stored XSS.

  1. Find a page that echoes input in the response.
  2. Find a page that stores and re-displays input.
  3. Verify execution with a harmless payload.
  4. Collect labeled XSS values.

Done when: Both XSS types work and labeled values are recorded.

Hints
Reflected XSS

The employee directory search reflects your query without encoding.

Stored XSS

The feedback form stores messages and renders HTML without sanitization.

XSS flag location

One labeled value is in the backup config from Phase 1, not on the search page.

Phase 4 — IDOR

Goal: Read another user's profile by changing an ID.

  1. Find an endpoint returning user profile data.
  2. Check whether ownership is validated.
  3. Enumerate IDs until you find sensitive admin fields.

Done when: You have a labeled value from another user's profile.

Hints
IDOR endpoint

Try /api/user/1, /api/user/2, /api/user/3…

Phase 5 — Local file read

Goal: Read a server file you should not access.

  1. Find a filename/path parameter.
  2. Try normal names, then traversal sequences.
  3. Collect labeled values from the response.

Done when: You extracted a labeled flag from a protected file.

Hints
File viewer

Look for /files?name=. Try filenames mentioned elsewhere first.

Phase 6 — Command injection

Goal: Run commands via a server-side utility.

  1. Find a diagnostic tool that runs shell commands.
  2. Inject metacharacters and read the output.
  3. Collect the labeled injection flag.

Done when: You have the command injection label from tool output.

Hints
Which tool

Network Tools passes input directly to a shell.

Injection characters

Append ;, |, &, backticks, or $() after a hostname.

Phase 7 — SSRF

Goal: Make the server fetch an internal URL.

  1. Find a server-side URL fetch feature.
  2. Identify internal-only endpoints from earlier recon.
  3. Pass the internal URL to the fetch tool.
  4. Read the labeled flag in the response.

Done when: You have the SSRF label from an internal endpoint.

Hints
URL preview tool

Linked from Network Tools. Accepts a url query parameter.

Internal target

Use INTERNAL_HEALTH_URL from the backup config so the server fetches it.

Phase 8 — Password reset

Goal: Approve a reset without the email flow.

  1. Generate a reset link for a known user.
  2. Determine whether the token is random or predictable.
  3. Forge a token for an admin and visit the reset URL.

Done when: You have the password reset label.

Hints
Token format

URL-safe base64 of the username — no secret key.

Phase 9 — Submission

Goal: Score 14/14 on the hidden form.

  1. Collect every labeled value from the site.
  2. Find the submission form (not linked in navigation).
  3. Enter each value under its matching label.
  4. Submit Yes for Stored XSS if your payload ran.

Done when: The form shows 14/14 complete.

Hints
Submission URL

/check

Bypass method values

sqli, sql injection, hidden role, role tamper, cookie, cookie tamper
